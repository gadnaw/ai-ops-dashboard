# Plan 01-03: DevOps — Vercel CI/CD, Pre-commit Hooks, ESLint, Prettier

## Objective

Lock down code quality and deployment pipeline. When this plan completes: every commit is blocked if it contains a `NEXT_PUBLIC_.*KEY` pattern (Pitfall 4 — API key exposure), ESLint and Prettier are enforced pre-commit, pushing to `main` triggers a Vercel production deployment, and the build process runs Prisma migrations before Next.js build (using `DIRECT_URL`).

This plan satisfies Phase 1 success criteria 2, 3, and 5.

### Task 2: Install Husky and lint-staged with pre-commit secret detection hook

**Wave:** 1
**Autonomous:** true
**Files:**
- `.husky/pre-commit`
- `.husky/commit-msg`
- `lint-staged.config.mjs`
- `.husky/_/husky.sh` (auto-generated by Husky)

**Implementation:**

Install Husky and lint-staged:
```bash
pnpm add -D husky lint-staged
pnpm exec husky init
```

This creates `.husky/pre-commit` with a default script. Replace with the full pre-commit hook.

**`.husky/pre-commit`** — the complete pre-commit hook:
```bash
#!/usr/bin/env sh
. "$(dirname -- "$0")/_/husky.sh"

# =============================================================================
# GUARD 1: Secret Detection (Pitfall 4 — API Key Exposure)
# =============================================================================
# Block any commit containing NEXT_PUBLIC_.*KEY patterns.
# LLM provider API keys prefixed with NEXT_PUBLIC_ are exposed to browsers.
# This is a hard stop — no bypass allowed.
#
# Pattern matches: NEXT_PUBLIC_OPENAI_KEY, NEXT_PUBLIC_ANTHROPIC_API_KEY, etc.

FORBIDDEN_PATTERN="NEXT_PUBLIC_[A-Z_]*KEY"
STAGED_FILES=$(git diff --cached --name-only)

if [ -n "$STAGED_FILES" ]; then
  MATCHES=$(git diff --cached | grep -E "^\+" | grep -E "$FORBIDDEN_PATTERN" || true)
  if [ -n "$MATCHES" ]; then
    echo ""
    echo "  ERROR: Secret Detection Failed"
    echo "  ─────────────────────────────────────────────"
    echo "  Detected NEXT_PUBLIC_*KEY pattern in staged changes."
    echo "  LLM provider API keys MUST NOT be prefixed with NEXT_PUBLIC_"
    echo "  because this exposes them in client-side JavaScript bundles."
    echo ""
    echo "  Offending lines:"
    echo "$MATCHES" | head -10
    echo ""
    echo "  Fix: Move the key to a server-only environment variable (no NEXT_PUBLIC_ prefix)."
    echo "  All LLM provider calls must go through server-side API routes."
    echo "  ─────────────────────────────────────────────"
    echo ""
    exit 1
  fi
fi

# =============================================================================
# GUARD 2: Lint-staged (ESLint + Prettier + TypeScript)
# =============================================================================
pnpm lint-staged
```

Make the hook executable:
```bash
chmod +x .husky/pre-commit
```

**`lint-staged.config.mjs`:**
```mjs
const config = {
  // TypeScript and JavaScript files — run ESLint fix and Prettier
  "**/*.{ts,tsx,js,jsx,mjs,cjs}": [
    "eslint --fix --max-warnings=0",
    "prettier --write",
  ],
  // JSON, CSS, and other formatting targets — Prettier only
  "**/*.{json,css,md}": ["prettier --write"],
};

export default config;
```

Add Husky setup to `package.json` scripts and `prepare` lifecycle:
```json
{
  "scripts": {
    "prepare": "husky"
  }
}
```

The `prepare` script runs automatically after `pnpm install`, ensuring all team members (or CI environments) get the hooks installed.

Test the secret detection hook:
```bash
# Create a test file with a forbidden pattern
echo 'const NEXT_PUBLIC_OPENAI_KEY = "sk-test"' > /tmp/test-secret.ts
git add /tmp/test-secret.ts
git commit -m "test: secret detection" 2>&1 | grep "Secret Detection"
# Expected: "ERROR: Secret Detection Failed"
# Clean up
git reset HEAD /tmp/test-secret.ts
rm /tmp/test-secret.ts
```

### Task 3: Configure Vercel deployment with build command and environment variables

**Wave:** 2
**Autonomous:** true
**Files:**
- `vercel.json`
- `.env.example` (update with Vercel-specific notes)

**Implementation:**

Create `vercel.json` — configures the Vercel build pipeline:
```json
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "framework": "nextjs",
  "buildCommand": "pnpm db:migrate && pnpm build",
  "installCommand": "pnpm install",
  "devCommand": "pnpm dev",
  "env": {
    "NEXT_TELEMETRY_DISABLED": "1"
  }
}
```

The `buildCommand` runs `prisma migrate deploy` (via `pnpm db:migrate`) before `next build`. This uses `DIRECT_URL` (direct connection, bypasses PgBouncer) to apply pending migrations in CI.

**Vercel project setup (execute these steps):**

1. Push the current codebase to GitHub (create repo if needed):
   ```bash
   git init
   git add -A
   git commit -m "feat(01-01,01-02,01-03): foundation scaffold, auth, and devops"
   git remote add origin https://github.com/USERNAME/REPO.git
   git push -u origin main
   ```

2. Install Vercel CLI and link project:
   ```bash
   pnpm add -g vercel
   vercel link
   ```
   Follow prompts: link to existing project or create new. Select the correct team/scope.

3. Add all environment variables to Vercel via CLI:
   ```bash
   # Production environment
   vercel env add DATABASE_URL production
   vercel env add DIRECT_URL production
   vercel env add NEXT_PUBLIC_SUPABASE_URL production
   vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY production
   vercel env add SUPABASE_SERVICE_ROLE_KEY production

   # Preview environment (can use same Supabase project for demo)
   vercel env add DATABASE_URL preview
   vercel env add DIRECT_URL preview
   vercel env add NEXT_PUBLIC_SUPABASE_URL preview
   vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY preview
   vercel env add SUPABASE_SERVICE_ROLE_KEY preview
   ```

4. Deploy to production:
   ```bash
   vercel --prod
   ```

5. Verify deployment:
   - Visit the deployment URL (e.g., `https://ai-ops-dashboard.vercel.app`)
   - Confirm `/` redirects to `/dashboard`
   - Confirm `/dashboard` redirects to `/login` (unauthenticated)
   - Confirm login page renders without errors

**Preview deployments on PRs:**

Preview deployments are automatic with Vercel. When a PR is opened against `main`, Vercel automatically builds and deploys a preview URL. No additional configuration needed.

To test: Create a branch, push it, open a PR — Vercel bot will comment with the preview URL.

## provides_interface Detail

### PreCommitHooks
- **Trigger:** `git commit` — runs automatically, no manual invocation
- **Secret detection pattern:** `NEXT_PUBLIC_[A-Z_]*KEY` (regex on staged diff)
- **Actions on match:** Exit 1 with actionable error message. No bypass path in normal workflow.
- **lint-staged targets:** `*.{ts,tsx,js,jsx,mjs,cjs}` → ESLint fix + Prettier; `*.{json,css,md}` → Prettier only
- **CI note:** `pnpm install` runs `prepare` script which installs Husky automatically

### VercelDeployment
- **Build command:** `pnpm db:migrate && pnpm build`
- **Migration behavior:** `prisma migrate deploy` runs using `DIRECT_URL` (port 5432 direct). Fails build if migration fails.
- **Preview deployments:** Automatic on all PRs, no configuration needed
- **Environment variables:** Set via `vercel env add` CLI or Vercel Dashboard
- **Domain:** Configure custom domain in Vercel Dashboard after initial deploy

### ESLintConfig
- **Config file:** `eslint.config.mjs` (flat config, not `.eslintrc`)
- **Key rules enforced:** `no-explicit-any`, `consistent-type-imports`, `no-unused-vars`
- **Extends:** `next/core-web-vitals`, `next/typescript`, `prettier` (disables style rules handled by Prettier)
- **Max warnings:** 0 in pre-commit (enforced by `--max-warnings=0` flag in lint-staged)

### PrettierConfig
- **Config file:** `.prettierrc.json`
- **Key settings:** 100-char print width, ES5 trailing commas, double quotes, Tailwind class sorting
- **Plugin:** `prettier-plugin-tailwindcss` sorts Tailwind classes automatically on save/commit
